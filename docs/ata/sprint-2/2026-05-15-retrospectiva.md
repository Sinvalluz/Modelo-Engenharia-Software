## 🎯 Objetivo da Retrospectiva

Analisar os avanços da Sprint 2, identificar dificuldades técnicas, registrar aprendizados e alinhar melhorias para continuidade do projeto.

---

# 1️⃣ Principais Entregas

- Criação inicial dos módulos backend:
  - Usuário (CRUD)
  - Categoria (CRUD)
  - Transação (CRUD)
- Estrutura inicial do banco relacional criada
- Continuidade da documentação do projeto
- Diagrama de classes finalizado
- Organização inicial das integrações entre frontend e backend
- Evolução das telas principais do sistema

---

# 2️⃣ Dificuldades Técnicas

- Dependências entre tarefas de frontend e backend impactaram continuidade de algumas demandas
- Algumas entregas técnicas chegaram incompletas para consumo de outras áreas
- Necessidade de maior alinhamento entre modelagem do banco e implementação prática
- Falta de Definition of Done mais clara para tarefas técnicas
- Hand-off entre membros precisa melhorar (entrega pronta para próximo colega continuar)
- Ajustes e mudanças próximas da entrega acabaram gerando retrabalho
- Necessidade de maior alinhamento técnico antes do desenvolvimento de algumas demandas

---

# 3️⃣ Lições Aprendidas

- Antes de mover tarefa para `Validation`, validar se ela realmente está utilizável pelo próximo responsável
- Comunicação antecipada evita gargalos entre backend e frontend
- Documentação precisa continuar sendo a principal referência técnica
- Refinamento técnico reduz retrabalho futuro
- Dependências entre tasks precisam ser melhor planejadas
- Revisões técnicas ajudam a evitar impactos inesperados no sistema

---

# 4️⃣ Oportunidades de Melhoria

- Criar checklist técnico para tasks backend:
  - Estrutura pronta
  - Migrations OK
  - Schema alinhado
  - Documentação mínima
- Melhor organização das dependências técnicas
- Refinar melhor tasks antes de iniciar desenvolvimento
- Melhorar continuidade entre entregas dos membros
- Centralizar melhor o backlog para reduzir confusão

---

# 5️⃣ Pendências / Itens Não Concluídos

- Adicionar informações de branches no Documento de Arquitetura
- Revisão completa do Documento de Arquitetura
- Ajustes finais na organização técnica do backend
- Melhor alinhamento entre modelagem e implementação prática
- Reorganização de algumas tasks no Kanban

---

# 6️⃣ Observações

- Sprint 2 apresentou aumento significativo da complexidade técnica
- Equipe conseguiu manter evolução constante mesmo com dependências entre áreas
- Próxima sprint será focada em estabilização, integração e conclusão das principais histórias do sistema
